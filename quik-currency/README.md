# quik-currency

`quik-currency` отвечает за валютный справочник и кросс-курсы, которые приходят из QUIK через таблицы MSSQL.

Сервис не публикует HTTP или gRPC API. Его публичное поведение сейчас состоит в изменении таблиц БД при запуске и в периодическом фоновом обновлении курсов.

## Что делает сервис

При запуске сервис:

1. Подключается к MSSQL.
2. Проверяет справочник `quik.currencies`.
3. Если справочник пустой, заполняет его активными валютами из ISO 4217.
4. Заполняет пустые `currency_name` по данным QUIK из `quik.current_quotes`.
5. Запускает периодический воркер переноса кросс-курсов из QUIK в `quik.fx_cbr_rates`.

## Таблицы

Сервис использует текущую схему `quik`.

Источник данных QUIK:

- `quik.current_quotes`.

Справочник валют:

- `quik.currencies`.

Таблица курсов:

- `quik.fx_cbr_rates`.

Таблицы внешних кодов и `ext_system_id` в текущем контуре не используются.

## Запуск

Из корня репозитория:

```bash
go run ./quik-currency/cmd
```

По умолчанию конфиг читается из:

```text
quik-currency/internal/configs/config.yaml
```

Путь к конфигу можно передать тем же механизмом `commonconfig`, который используется в остальных сервисах репозитория.

## Конфигурация

В конфиге нужны блоки:

- `log` - уровень, формат и файл логов;
- `db` - подключение к MSSQL.

## Инициализация валют

Если `quik.currencies` пустая, сервис заполняет ее активными валютами из ISO 4217.

После этого сервис обновляет пустые `currency_name` по котировкам QUIK:

- источник: `quik.current_quotes`;
- фильтр: `class_code = 'CROSSRATE'`;
- коды `SUR`, `RUR`, `RUB` нормализуются в `RUB`;
- новые валюты из QUIK автоматически не добавляются.

Если валюта есть в `current_quotes`, но отсутствует в `quik.currencies`, она игнорируется на этапе заполнения имени.

## Перенос кросс-курсов

Фоновый воркер раз в `60*time.Second` переносит кросс-курсы из `quik.current_quotes` в `quik.fx_cbr_rates`.

Основные правила:

- берутся строки `class_code = 'CROSSRATE'`;
- курс берется из `COALESCE(close_price, last_price)`;
- коды `RUR`, `SUR`, `RUB` не переносятся как базовая валюта;
- валюта должна существовать в `quik.currencies`;
- `quote_iso_code` для записей в `quik.fx_cbr_rates` равен `643` (RUB);
- повторный запуск не создает дубли по ключу `(date, quote_iso_code, base_iso_code)`.

Если код из QUIK не найден в `quik.currencies`, курс не переносится, а в лог пишется `warn` с датой и кодом валюты.

## Ограничения

- Сервис не предоставляет публичный HTTP/gRPC API.
- Новые валюты из QUIK автоматически не добавляются в `quik.currencies`.
- Интервал воркера задан в коде.
- Несопоставленные коды логируются при каждом цикле воркера, пока не появятся в справочнике или не будут обработаны отдельным правилом.
